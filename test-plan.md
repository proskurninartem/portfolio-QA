# Тест-план: Reqres.in REST API

**Версия:** 1.0  
**Автор:** Артём Проскурнин  
**Дата:** апрель 2026  
**Статус:** Выполнен

---

## 1. Цели тестирования

- Проверить корректность работы REST API эндпоинтов (CRUD)
- Убедиться в соответствии ответов API документации (контракт)
- Выявить дефекты в обработке граничных значений и негативных сценариев
- Проверить коды ответов, структуру JSON и заголовки ответов

---

## 2. Объём тестирования (Scope)

### В рамках тестирования:
- `GET /api/users` — список пользователей с пагинацией
- `GET /api/users/{id}` — получение одного пользователя
- `POST /api/users` — создание пользователя
- `PUT /api/users/{id}` — полное обновление пользователя
- `PATCH /api/users/{id}` — частичное обновление
- `DELETE /api/users/{id}` — удаление пользователя
- `POST /api/login` — авторизация
- `POST /api/register` — регистрация

### Вне рамок тестирования:
- Нагрузочное и стресс-тестирование
- Тестирование безопасности (penetration testing)
- Автоматизация (ручное тестирование)

---

## 3. Виды тестирования

| Вид тестирования | Описание |
|---|---|
| Функциональное | Соответствие поведения API требованиям |
| Позитивное | Корректные запросы → ожидаемые ответы |
| Негативное | Некорректные данные → корректная обработка ошибок |
| Граничные значения | Минимальные/максимальные значения параметров |
| Контрактное | Структура JSON-ответа соответствует схеме |

---

## 4. Критерии начала тестирования

- API доступен и возвращает ответы
- Документация API актуальна
- Инструменты для тестирования настроены (Postman)

---

## 5. Критерии завершения тестирования

- Выполнено 100% запланированных тест-кейсов
- Все дефекты приоритета Critical/High зафиксированы
- Подготовлен отчёт по результатам

---

## 6. Риски

| Риск | Вероятность | Влияние | Митигация |
|---|---|---|---|
| API недоступен | Низкая | Высокое | Повторное выполнение, mock-сервер |
| Изменение API без уведомления | Средняя | Среднее | Проверка документации перед тестами |
| Нестабильные тестовые данные | Средняя | Среднее | Использование фиксированных ID |

---

## 7. Тестовое окружение

| Параметр | Значение |
|---|---|
| Base URL | `https://reqres.in` |
| Инструмент | Postman v10+ |
| Формат данных | JSON |
